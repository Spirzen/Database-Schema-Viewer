## О проекте

**Database Schema Viewer** — десктопное веб-приложение для разработчиков и аналитиков, которым нужно быстро понять структуру чужой или своей базы: таблицы, столбцы, первичные и внешние ключи, направление связей.

В отличие от встроенных диаграмм в SSMS, pgAdmin или DBeaver, здесь акцент на:

- **читаемой интерактивной схеме** (масштаб, панорама, подсветка связанных таблиц);
- **понятных формулировках** («поле X таблицы A связано с полем Y таблицы B»);
- **лёгком старте** — демо SQLite без установки сервера;
- **мягком неоновом UI** с тёмной и светлой темой.

Приложение состоит из локального **Python API** (чтение метаданных) и **React-клиента** (визуализация). Данные не отправляются на внешние сервера — всё работает на вашем компьютере.

## Возможности

### Подключение и обнаружение

| Функция | Описание |
|--------|----------|
| **5 СУБД** | SQLite, PostgreSQL, MySQL, MS SQL Server, Oracle |
| **Предзаполнение** | `localhost` и стандартные порты (5432, 3306, 1433, 1521) |
| **Список баз** | Обновление списка БД + ручной ввод имени |
| **Получить сервера** | Сканирование локальных экземпляров PostgreSQL, MySQL, MS SQL (порты, службы Windows, реестр, SQL Browser) |
| **Проверка подключения** | Тест до загрузки схемы |

### Диаграмма

| Функция | Описание |
|--------|----------|
| **ER-граф** | Таблицы как узлы, FK как рёбра с подписями |
| **Динамическая ширина узлов** | Блок таблицы растягивается под длину имени и столбцов |
| **Клик по таблице** | Карточка с вкладками «Поля» и «Связи» |
| **Клик по связи** | Текстовое описание FK и переход к таблицам |
| **Подсветка** | Выбранная таблица, все связанные с ней, яркие линии FK |
| **Поиск** | По имени таблицы или столбца с подсветкой на схеме |
| **Фильтр таблиц** | Виртуализированный список, «Все» / «Связанные» / выборочно |
| **Переложить** | Повторная автоматическая раскладка (dagre / сетка) |
| **Экспорт PNG** | Снимок текущего вида диаграммы |
| **Minimap** | При умеренном числе таблиц |

### Карточка таблицы

- Вкладка **«Поля»** — компактный список столбцов, типов, меток PK / FK / NOT NULL.
- Вкладка **«Связи»** — исходящие и входящие FK с пояснениями и кнопками перехода.
- Кнопки **«Диаграмма»** и переход к связанной таблице / полю.
- Адаптивная панель: на широком экране — справа поверх графа, на узком — снизу как шторка.

### Прочее

- Тёмная и светлая тема (сохранение в `localStorage`).
- Демо-база SQLite в репозитории (7 таблиц, 8 связей).
- Оптимизация для **1000+ таблиц** (компактный режим, отрисовка только видимых узлов, лимит на «показать все»).

## Архитектура

```
┌─────────────────────────────────────────────────────────────────┐
│  Браузер  http://localhost:5173                                  │
│  ┌──────────────┐  ┌────────────────────────┐  ┌─────────────┐ │
│  │ Connection   │  │  React Flow (схема)     │  │  Details    │ │
│  │ Panel        │  │  + Toolbar + Filter     │  │  Panel      │ │
│  └──────┬───────┘  └───────────┬────────────┘  └─────────────┘ │
│         │                      │ proxy /api                     │
└─────────┼──────────────────────┼────────────────────────────────┘
          │                      ▼
          │            ┌─────────────────────┐
          └───────────►│  FastAPI :18765      │
                       │  connectors/         │
                       │  discovery.py        │
                       └──────────┬──────────┘
                                  │
                    ┌─────────────┼─────────────┐
                    ▼             ▼             ▼

```

1. Пользователь задаёт параметры подключения во frontend.
2. Frontend вызывает REST API (`/api/schema`, `/api/databases`, …).
3. Backend выбирает коннектор по `db_type`, читает метаданные, возвращает JSON.
4. Frontend строит узлы и рёбра, раскладывает граф, отображает карточки.

## Быстрый старт

### Вариант A — скрипт Windows

```powershell
cd "K:\Database Schema Viewer"
.\start.ps1
```

Скрипт создаёт демо-БД (если нет), venv, ставит зависимости, запускает backend и frontend.

Откройте **http://localhost:5173** → нажмите **«Демо SQLite»**.

### Вариант B — вручную

**1. Демо-база**

```powershell
python sample\create_demo_db.py
```

Создаётся `sample\demo.db` (отделы, сотрудники, заказы, клиенты…).

**2. Backend**

```powershell
cd backend
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
python main.py
```

API: **http://127.0.0.1:18765**  
Документация Swagger: **http://127.0.0.1:18765/docs**

**3. Frontend** (новый терминал)

```powershell
cd frontend
npm install
npm run dev
```

Откройте **http://localhost:5173**.

### Смена порта API

```powershell
$env:DSV_PORT = "18800"
python main.py
```

В `frontend/vite.config.ts` proxy должен указывать на тот же порт.

### Production-сборка frontend

```powershell
cd frontend
npm run build
npm run preview
```

Backend по-прежнему нужен для работы с БД.

## Подключение к СУБД

| СУБД | Хост | Порт | База / файл | Схема |
|------|------|------|-------------|--------|
| **PostgreSQL** | `localhost` | `5432` | из списка ↻ | `public` (по умолчанию) |
| **MySQL** | `localhost` | `3306` | из списка ↻ | — |
| **MS SQL Server** | `ИМЯ_ПК\SQLEXPRESS` или `localhost` | `1433`* | из списка ↻ | `dbo` |
| **Oracle** | `localhost` | `1521` | Service / SID | owner (часто = пользователь) |
| **SQLite** | — | — | путь к `.db` | — |

\* Для **именованного экземпляра** (`SPIRZEN\SQLEXPRESS`) указывайте только хост **без порта** — поле порта скрывается.

### Советы

- Сначала **«Проверить»**, затем **↻** для списка баз, затем **«Подключить»**.
- **«Получить сервера»** — если не знаете адрес локального PostgreSQL / MySQL / SQL Server.
- В поле хоста не вводите `http://` — приложение очистит URL автоматически.

## Производительность

При большом числе таблиц (сотни и тысячи):

| Порог | Поведение |
|-------|-----------|
| **> 40 видимых** | Компактные узлы (только имя таблицы) |
| **> 120** | Минимальный компактный режим |
| **> 50** | Раскладка сеткой вместо dagre |
| **При загрузке** | На диаграмму сразу выводятся связанные таблицы (до 200), не все |
| **«Все» в фильтре** | Не более 500 таблиц за раз |
| **Отрисовка** | `onlyRenderVisibleElements` — рисуются только узлы в viewport |

## API backend

| Метод | Путь | Описание |
|-------|------|----------|
| `GET` | `/api/defaults` | localhost, порты по СУБД |
| `GET` | `/api/sample-path` | Путь к демо `demo.db` |
| `GET` | `/api/discover-servers` | Поиск локальных серверов |
| `POST` | `/api/test` | Проверка подключения |
| `POST` | `/api/databases` | Список баз |
| `POST` | `/api/schema` | Таблицы, столбцы, FK |

Тело запроса для подключения (пример):

```json
{
  "db_type": "postgresql",
  "host": "localhost",
  "port": 5432,
  "username": "postgres",
  "password": "***",
  "database": "mydb",
  "schema": "public",
  "file_path": ""
}
```

## Устранение неполадок

| Проблема | Решение |
|----------|---------|
| `WinError 10013` при запуске API | Порт занят. Задайте `$env:DSV_PORT = "18800"` и обновите proxy в `vite.config.ts` |
| PostgreSQL «висит» на подключении | Убедитесь, что выбрана база; большие схемы грузятся до 2 мин |
| MySQL `10061` | Сервер не запущен на `localhost:3306` |
| MS SQL таймаут | Проверьте `ПК\ЭКЗЕМПЛЯР`, TCP/IP в конфигурации SQL Server |
| Пустой список серверов | Запустите СУБД; для SQL — проверьте службу и SQL Browser |
| Диаграмма пустая | Откройте фильтр → «Связанные» или отметьте таблицы |
| Frontend не видит API | Backend должен работать на том же порту, что proxy (18765) |
